# CanopyRS HPC Windows Setup Guide

**Official CanopyRS GitHub Repository:** [https://github.com/hugobaudchon/CanopyRS](https://github.com/hugobaudchon/CanopyRS)

This documentation will guide you step by step through installing and configuring CanopyRS on a Windows High-Performance Computing (HPC) machine. You can run these commands in the Windows Command Prompt (`cmd`). 

## Project Directory Structure

Before we begin, let's understand where all the files will go. We will assume you are working in a main directory called `D:\YourFolderName`. By the end of this setup, your folder structure will look like this:

```text
D:\YourFolderName\
 ├── CanopyRS\                     # The main cloned CanopyRS repository
 │    ├── detrex\                  # Object detection submodule
 │    └── canopyrs\                # Core scripts and configurations
 │
 ├── envs\
 │    └── canopyrs\                # Your isolated Conda Python environment
 │
 ├── BuildTools\                   # Microsoft Visual Studio C++ Build Tools
 │
 ├── cuda_main\                    # Local CUDA Toolkit installation
 │    ├── bin\                     # Contains the nvcc compiler
 │    ├── include\                 
 │    └── lib\                     
 │
 ├── checkpoints\                  # Folder containing manually downloaded AI model weights
 │    ├── dino-detector\           # Contains model_best.pth
 │    ├── sam3-ft\                 # Contains sam3_selvamask_ft_model_best.pt
 │    └── facebook-sam3\           # Contains base SAM3 model files
 │
 ├── bci-crown-model\              # Species detection AI model (transferred from local PC)
 │
 ├── powai_files\                  # Directory holding your input data
 │    ├── Academic_area_clip.tif  
 │    └── Tree_canopy_academic_area.gpkg 
 │
 ├── p1\                           # The main output folder generated by the CanopyRS pipeline
 │    ├── 0_tilerizer\             # Step 1: Image sliced into tiles
 │    ├── 1_detector\              # Step 2: DINO finds bounding boxes
 │    ├── 2_aggregator\            # Step 3: Merges duplicate boxes
 │    ├── 3_segmenter\             # Step 4: SAM3 generates precise polygons
 │    ├── 4_aggregator\            # Step 5: Final aggregated raw map polygons (.gpkg). THIS IS THE MAIN GPKG FILE THAT WE                                                WILL USE
 │    ├── 5_tilerizer\             # Step 6: Individual cutouts of each crown
 │    └── crowns_with_species.gpkg # THE FINAL RESULT: Map with predicted species attached!
 │
 ├── images\                       # Species detection graphs 
 │    ├── chart_7_5_species_distribution.png
 │    └── chart_7_6_per_crown.png
 │
 ├── accuracy_score.py             # Script to calculate model accuracy 
 └── Species_detection_final_results_download.py  # Script for final species processing
```

## Step 1: Network Check and Cloning the Repository

First, open your Command Prompt (`cmd`) and navigate to your main drive (e.g., `D:`). We need to check the network proxy settings and download the CanopyRS repository.

```cmd
Run the commands that are inside this 
```

**1. unset your user-level proxy:**

```cmd
git config --global --unset http.proxy
git config --global --unset https.proxy
# If it says "no such section" for one of them, that's fine — ignore and continue.
```

**2. Test internet connection to GitHub:**

```cmd
curl -v https://github.com
```

*Explanation: This pings GitHub to ensure the HPC has the necessary internet access to download the code.*

**3. Download the CanopyRS code:**

```cmd
git clone https://github.com/hugobaudchon/CanopyRS.git
```

*Explanation: This downloads the entire CanopyRS codebase from GitHub to your machine.*

**4. Enter the CanopyRS folder:**

```cmd
cd CanopyRS
```

*Explanation: This changes your current directory so you are inside the downloaded CanopyRS folder.*

## Step 2: Setting up the Conda Environment

We need to create an isolated "environment" for CanopyRS so its dependencies do not interfere with other projects on the HPC.

**1. Find where Conda is installed:**

```cmd
where conda
dir C:\ProgramData\Miniconda3\Scripts\conda.exe
dir C:\ProgramData\Anaconda3\Scripts\conda.exe
dir "%USERPROFILE%\Miniconda3\Scripts\conda.exe"
dir "%USERPROFILE%\Anaconda3\Scripts\conda.exe"
```

*Explanation: These commands search the computer to find the exact location of the Anaconda/Miniconda installation and give its exact path in the output
for eg C:\Users\admin\Anaconda3\Scripts\activate.bat*

**2. Activate the base Conda environment:**
*(Note: Adjust this path if step 1 showed Anaconda is installed elsewhere)*

```cmd
call C:\Users\admin\Anaconda3\Scripts\activate.bat
```

*Explanation: This turns on the base Conda package manager so we can use it to create our custom environment.*

**3. Check Conda version:**

```cmd
conda --version
```

*Explanation: Verifies that Conda is activated and working.*

**4. Create the new CanopyRS environment:**

```cmd
conda create --prefix D:\YourFolderName\envs\canopyrs -c conda-forge python=3.10 mamba
```

*Explanation: Creates an isolated environment folder at `D:\YourFolderName\envs\canopyrs` using Python version 3.10, and installs `mamba` (a faster version of Conda).*

**5. Activate the new environment:**

```cmd
conda activate D:\YourFolderName\envs\canopyrs
```

*Explanation: Switches your terminal to use the newly created environment. 



#### **Important**

 You must do this every time you open a new terminal to activate our environment .*

```cmd
call C:\Users\admin\Anaconda3\Scripts\activate.bat
conda activate D:\Tanushree\envs\canopyrs
```

## Step 3: Installing Core Packages (GDAL and PyTorch)

**1. Install GDAL using Mamba:**

```cmd
mamba install gdal=3.6.2 -c conda-forge
```

*Explanation: GDAL is a crucial library for reading spatial/mapping data.*

**2. Configure network and channel settings:**

```cmd
conda config --set ssl_verify true
mamba config --set ssl_verify true
mamba clean -a
conda config --set channel_priority strict
```

*Explanation: These commands enforce strict security (SSL) for downloads, clean up leftover cache files to prevent errors, and force Conda to prioritize the `conda-forge` channel for packages.*

**3. Re-verify GDAL installation via Conda:**

```cmd
conda install gdal=3.6.2 -c conda-forge
```

*Explanation: Ensuring GDAL is strictly locked and properly registered in the environment.*

**4. Install PyTorch (GPU version):**

```cmd
pip install torch==2.7.1 torchvision==0.22.1 --index-url https://download.pytorch.org/whl/cu126
```

*Explanation: Installs the AI machine learning library (PyTorch) configured specifically for NVIDIA GPUs (CUDA 12.6).*

**5. Test if the GPU is detected:**

```cmd
python -c "import torch; print('CUDA available:', torch.cuda.is_available()); print('Device:', torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'CPU only')"
```

*Explanation: This is a test. If it prints `CUDA available: True` and names the NVIDIA GPU, your setup is correct.*

**6. Update CanopyRS submodules and install:**

```cmd
git submodule update --init --recursive
python -m pip install -e .
```

*Explanation: CanopyRS relies on other repositories (submodules). This downloads them and installs CanopyRS in "editable" mode.*

## Step 4: Installing Visual Studio Build Tools

To compile specific parts of the AI models, we need C++ build tools.

**1. Go back to your main folder:**

```cmd
cd /d D:\YourFolderName
```

**2. Download the Build Tools installer:**

```cmd
curl -L -o vs_BuildTools.exe https://aka.ms/vs/17/release/vs_BuildTools.exe
```

*Explanation: Downloads the official Microsoft Visual Studio Build Tools installer.*

**3. Install the Build Tools silently:**

```cmd
vs_BuildTools.exe --quiet --wait --norestart --nocache --installPath "D:\YourFolderName\BuildTools" --add Microsoft.VisualStudio.Workload.VCTools --includeRecommended
```

*Explanation: Runs the installer in the background without pop-ups and installs it directly into `D:\YourFolderName\BuildTools`. This is a ~4GB download and may take a few minutes.
There will be no output here*

**4. Check if the installer is still running:**

```cmd
tasklist | findstr /i "vs_installer setup BuildTools"
```

*Explanation: If this returns some text, the installation is still processing. Wait until it finishes.*

**5. Verify the folder exists:**

```cmd
dir "D:\YourFolderName\BuildTools" 2>nul
```

*Explanation: Checks if the folder was successfully created. (`2>nul` just hides error messages if it doesn't exist yet).
Or you can check directly in File Explorer*

**6. Re-run PyTorch install (to ensure paths are bound):**

```cmd
pip install torch==2.7.1 torchvision==0.22.1 --index-url https://download.pytorch.org/whl/cu126
```

## Step 5: Compiling Detectron2 and Detrex

This step compiles the core object detection libraries. You will see some test commands fail at first—this is expected, and the subsequent commands will fix them.

**1. Activate the Build Tools compiler:**

```cmd
call "D:\YourFolderName\BuildTools\VC\Auxiliary\Build\vcvars64.bat"
```

*Explanation: This temporarily loads the Microsoft C++ compiler into your terminal session.*

**2. Verify the compiler is active:**

```cmd
where cl
```

*Explanation: If successful, this will output the path to `cl.exe` (the compiler).*

**3. Navigate to CanopyRS and compile Detrex/Detectron2:**

```cmd
cd /d D:\YourFolderName\CanopyRS
python -m pip install --no-build-isolation -e ./detrex/detectron2 -e ./detrex
set DISTUTILS_USE_SDK=1
python -m pip install --no-build-isolation -e ./detrex/detectron2 -e ./detrex
```

*Explanation: We enter the directory, trigger the compilation of the AI libraries, set a flag telling Python to use the compiler we just activated, and run the compilation again to ensure it builds correctly.*

Here are the two steps with **Step 5** moved before **Step 4**:

---

**4. The `_C` extension error fix:**  

```cmd
python -c "from detectron2 import _C; print('detectron2 _C extension: OK')"
set PATH=D:\YourFolderName\envs\canopyrs\Lib\site-packages\torch\lib;%PATH%
pip install pefile
```

*Explanation: Setting the PATH links the missing PyTorch library files, and installing `pefile` resolves Windows executable parsing errors.*

**5. Test the installation**

```cmd
python -c "import torch, detectron2, detrex; print('torch:', torch.__version__); print('CUDA:', torch.cuda.is_available()); print('detectron2:', detectron2.__version__); print('detrex: OK')"

python -c "import torch, detectron2; print('torch:', torch.__version__); print('CUDA:', torch.cuda.is_available()); print('detectron2:', detectron2.__version__)"

python -c "import canopyrs; print('CanopyRS installed successfully')"
```

*Explanation: These are test scripts to check if the modules load.*

**6. Final path fixes:**

```cmd
python -c "import detrex, os, glob; print(glob.glob(os.path.join(os.path.dirname(detrex.__file__), '_C*.pyd'))[0])"
python -c "import detrex; print('Path:', detrex.__path__); print('File:', detrex.__file__)"
cd /d D:\YourFolderName
python -c "import detrex; print('Path:', detrex.__path__); print('File:', detrex.__file__)"
```

*Explanation: These commands verify that Python is successfully locating the compiled `_C.pyd` files now that `pefile` is installed and the directories are set.*

## Step 6: Hugging Face Setup and Downloading Models

CanopyRS uses large AI models hosted on Hugging Face. We need to log in and download them locally.

### 1. Initial Setup

1. Create a free Hugging Face account on your laptop(not on the HPC) if you don't already have one:
   
   * [https://huggingface.co/join](https://huggingface.co/join)

2. After signing in, open the following model page:
   
   * [https://huggingface.co/facebook/sam3](https://huggingface.co/facebook/sam3)

3. Fill out the access request form and accept Meta's license terms.

4. Wait for your request to be approved.
   
   * Approval is automatic and usually takes a few minutes (up to about 10 minutes).
   * **Do not continue until your request has been approved.**
   * You can refresh the model page to check whether you have access.

### 2. Downloading the AI Models (On Your Local Computer)

Important: The HPC server has a strict firewall that blocks downloading large AI files directly from the internet. You must do this entire step on your personal local computer/laptop, and then transfer the files to the HPC.

1. Download the Base SAM3 Files:
   
    Go to the files page: https://huggingface.co/facebook/sam3/tree/main
   
    You must download every single file listed on this page (there are about 13 files, including a large ~3.4 GB model.safetensors file).
   
    Save all these files in a folder named exactly facebook-sam3. 

2. Download the CanopyRS DINO Detector:
   
    Go to: https://huggingface.co/CanopyRS/dino-swin-l-384-multi-NQOS-selvamask-FT/tree/main
   
    Click the download icon next to the file named model_best.pth

3. Download the CanopyRS SAM3 Fine-Tuned Segmenter:
   
    Go to: https://huggingface.co/CanopyRS/sam3-multi-selvabox-selvamask-FT/tree/main
   
    Click the download icon next to the file named sam3_selvamask_ft_model_best.pt

4. Transfer Everything to the HPC:
   
    Copy the facebook-sam3 folder and the two .pth / .pt files.
   
    Connect to the HPC and paste them directly into your main project folder (e.g.,   D:\YourFolderName).

## Step 7: Organizing Model Checkpoint Files

Goal: We need to put the downloaded model files into a structured `checkpoints` folder so CanopyRS knows exactly where to find them offline.

**1. Create the checkpoint directories:**

```cmd
mkdir checkpoints 2>nul
mkdir checkpoints\dino-detector 2>nul
mkdir checkpoints\sam3-ft 2>nul
```

*Explanation: `mkdir` creates new folders. Adding `2>nul` just hides any error messages in case the folders already exist.*

**2. Move the model weight files:**
*(Note: Ensure you have already downloaded the `model_best.pth` and `sam3_selvamask_ft_model_best.pt` files into your main folder before running this).*

```cmd
move D:\YourFolderName\model_best.pth D:\YourFolderName\checkpoints\dino-detector\
move D:\YourFolderName\sam3_selvamask_ft_model_best.pt D:\YourFolderName\checkpoints\sam3-ft\
```

*Explanation: This moves the individual AI weight files into their specific subfolders.*

**3. Move the SAM3 folder:**

```cmd
move D:\YourFolderName\facebook-sam3 D:\YourFolderName\checkpoints\
```

*Explanation: Moves it inside the `checkpoints` folder.*

## Step 8: Updating Configuration Files

Now that the files are local, we must edit the configuration files to point to our local `D:` drive instead of the internet. You can use Notepad or any text editor for this.

**1. Edit the DINO configuration file:**

```cmd
notepad D:\YourFolderName\CanopyRS\canopyrs\config\detectors\dino_swinL_multi_NQOS_selvamask_FT.yaml
```

*Action:* Find the `checkpoint_path:` line (around the 3rd line).
*Replace:* `https://huggingface.co/CanopyRS/dino-swin-l-384-multi-NQOS-selvamask-FT/resolve/main/model_best.pth`
*With:* `checkpoint_path: D:/YourFolderName/checkpoints/dino-detector/model_best.pth`
*Explanation: This tells the DINO model to load from your local hard drive.*

**2. Edit the SAM3 configuration file:**

```cmd
notepad D:\YourFolderName\CanopyRS\canopyrs\config\segmenters\sam3_multi_selvamask_FT.yaml
```

*Action:* Find the `checkpoint_path:` line.
*Replace the URL with:* `checkpoint_path: D:/YourFolderName/checkpoints/sam3-ft/sam3_selvamask_ft_model_best.pt`
*Explanation: This points the segmentation model to your local files.*

**3. Edit the Python script for SAM3:**

```cmd
notepad D:\YourFolderName\CanopyRS\canopyrs\engine\models\segmenter\sam3.py
```

*Action:* Find and replace all text that says `"facebook/sam3"` with the full local path: `"D:/YourFolderName/checkpoints/facebook-sam3"`. Save the file.
*Explanation: This prevents the Python code from trying to connect to the internet to find the base SAM3 model.*

## Step 9: Local CUDA Toolkit Installation & Recompiling

**1. Download CUDA 12.6:**
Go to the NVIDIA CUDA Download Archive:
[https://developer.nvidia.com/cuda-12-6-0-download-archive?target_os=Windows&target_arch=x86_64&target_version=Server2022&target_type=exe_local](https://developer.nvidia.com/cuda-12-6-0-download-archive?target_os=Windows&target_arch=x86_64&target_version=Server2022&target_type=exe_local)

* Select: OS Windows -> Architecture x86_64 -> Version Server 2022 -> Installer Type exe (local).

**2. Install CUDA:**

* Double-click the installer.
* Choose a custom location: `D:\YourFolderName\cuda_main`.
* **Important:** During installation, check **ONLY** these three boxes: Development, Runtime, and Compiler. Uncheck everything else.

**3. Link CUDA and Recompile Detrex:**

```cmd
call D:\YourFolderName\BuildTools\VC\Auxiliary\Build\vcvars64.bat
set CUDA_HOME=D:\YourFolderName\cuda_main
cd D:\YourFolderName\CanopyRS\detrex
set DISTUTILS_USE_SDK=1
pip install -e . --no-build-isolation
cd \YourFolderName
```

*Explanation: This tells Windows where the newly installed CUDA compiler is, and recompiles the `detrex` library to fully utilize your specific local GPU environment.*

## Step 10: Running the CanopyRS Automation Pipeline

*(Note: Whenever you close and reopen the terminal, you must always run `call C:\Users\admin\Anaconda3\Scripts\activate.bat` and `conda activate D:\YourFolderName\envs\canopyrs` before doing anything).*

*(Note: Adding 5th step is optional only needed when you need to do species detection)*

**1. Update the Preset File (Adding the 5th Step):**
Open your preset file located at YourFolderName/CanopyRS/canopyrs/config/pipelines (`preset_seg_multi_NQOS_selvamask_SAM3_FT_quality.yaml`) in a text editor. Scroll to the very end and paste the following block to add the "tilerizer" step:

```yaml
  - tilerizer:
      tile_type: polygon
      tile_size: 512
      ground_resolution: 0.019
      use_variable_tile_size: false
```

Save and close the file.

**2. Run the main automation command:**
Run the pipeline on the academic area image. 
*(File Size: `3.4 GB`)*
*(Expected Time to Finish: `15-20 min`)
*Expected time depends on the size of the tif file *

*

```cmd
python D:\YourFolderName\CanopyRS\infer.py -c preset_seg_multi_NQOS_selvamask_SAM3_FT_quality.yaml -i D:\YourFolderName\CanopyRS\assets\[Academic Area Placeholder].tif -o D:\YourFolderName\p1
```

*Explanation: This is the main command that runs the AI models to detect and map the tree crowns in the `.tif` image. It uses the preset file we just modified and outputs all results into a new folder called `p1`.*

**Note on Errors during execution:** Because the HPC does not have an internet connection, you will see `UserWarnings` and lines that look like errors scrolling on the screen. **You can completely ignore these.** The software is just complaining about being offline.

**3. Check your results:**
If the process is successful, the very last line in the terminal will say:
` ------ Pipeline finished ------ `

Navigate to `D:\YourFolderName\p1\4_aggregator`. You will find two `.gpkg` files. The final map file you need is the one that does **NOT** have "nonaggregated" in its name.

## Step 11: Downloading the Species Detection Model

To identify the tree species, we need a specific AI model. Since the HPC might have restricted internet access, it is easiest to download this model on your local computer first, and then transfer it to the HPC.

**1. On your local computer (not the HPC), open a terminal and run:**

```cmd
pip install huggingface_hub
```

*Explanation: This installs the tool needed to download files from Hugging Face onto your personal computer.*

**2. Download the BCI crown model:**

```cmd
python -c "from huggingface_hub import snapshot_download; snapshot_download('sulagnasaharasha/bci-crown-model', local_dir=r'C:\Users\YourName\Downloads\bci-crown-model')"
```

*Note: Make sure to change `YourName` to your actual Windows username in the path above.*
*Explanation: This command downloads the pre-trained species detection model directly into your local Downloads folder.*

**3. Transfer the model to the HPC:**
Copy the entire `bci-crown-model` folder from your local computer and paste it into the main directory on the HPC (e.g., `D:\YourFolderName\bci-crown-model`).

# Suggestions for Future Work

These are notes from my attempts to improve accuracy.

1. Fine-tuning DINO Swin-L - Training got stuck at 100% memory usage. Would need a 24 GB+ GPU.

2. Fine-tuning DINO ResNet-50 (default learning rate 0.0001) - Training completed, but very few trees detected. Learning rate too high — the model lost its pre-trained knowledge.

3. Fine-tuning DINO ResNet-50 (reduced learning rate 0.00001) -  Metrics stabilised, but the model detected only 450 of 1,850 real trees on the full image.

Official CanopyRS NMS grid search tool. Requires the dataset to be registered in CanopyRS's internal dataset registry. 

Suggestions to try

1. The detector is the weak part of the pipeline. A mean IoU of 73% shows that when the detector finds a tree, SAM 3 draws the polygon shape well, so the segmentor is good.

2. Try SAM 2 as an alternative segmenter. We used SAM 3. SAM 2 may give slightly different results on this data as this was used in Google Colab and results were better there.

3. Targeting fine-tuning on missed trees. Instead of fine-tuning on all 1,850 annotations, identify the trees the current model misses (false negatives from accuracy_score.py output), extract just those from the manual GeoPackage, and fine-tune on that focused subset. This might work.

4. Read the official CanopyRS docs properly.
